# Грабли — снейл «уезжает за правый край» на телефоне (удвоение ширины)

Дата: 2026-06-09. Устройство: Redmi Note 10 Pro (`sweet`, 1080×2400 @ 440dpi → ppp 2.75).

## Симптом
В Android-сборке круглый снейл центрировался **ровно на правом крае экрана** —
видна была только левая половина круга (метки `G,Ab,A,Bb,B`), а центр и правая
половина (`C,Db,D,Eb,E,F,Gb`) уезжали за экран. Полоса настроек (`Spread/Glow/…`)
тоже убегала вправо. Ключевая примета: центр строго на крае ⇒ **ширина ровно
удвоена**, это не «чуть-чуть не влезло».

## Чего НЕ было (отметено замерами)
- Не ориентация. `orientation="portrait"` отрабатывает, экран действительно
  портретный.
- Не DPI/ppp. `screen_rect = 392.7 dp` = 1080/2.75 — корректно.
- Не «снейл — круг, надо квадратить высоту». Распирало именно **ширину**; высота
  и подзаголовки — симптомы соседние, корень не в них.

## Как диагностировали (переиспользуемая техника)
`log`-крейт на Android молчит (см. KICKOFF, грабли №2) → послойный дамп через
`crate::android_perm::alog(...)`, разово (`AtomicBool::swap`), тег `snail`,
смотреть `adb logcat -s snail`. Замеряли `ui.max_rect()` на каждом слое сверху вниз:

| точка | ширина `max_rect` |
|---|---|
| `screen_rect` (экран) | 392.7 dp ✅ |
| верх карточки (до `Frame`) | 376.7 dp ✅ |
| **после хедера** | 346.7 dp ✅ |
| **после блока настроек** | **813.7 dp** 💥 |
| аллокация снейла | 813.7 ⇒ центр на ~418 = правый край |

Удвоение локализовалось **строго между «после хедера» и «после настроек»** ⇒
виновник — `draw_mobile_snail_settings`.

## Корень
Ручки настроек были обёрнуты в `ui.horizontal_wrapped`, а **каждая** ручка — это
вложенный фиксированный `ui.horizontal` (label + slider + readout):

```rust
ui.horizontal_wrapped(|ui| {
    mobile_slider(ui, "Spread", ...); // внутри: ui.horizontal(|ui| { … })
    mobile_slider(ui, "Glow",   ...);
    …
});
```

egui при вложении **не-переносящего `horizontal` внутрь `horizontal_wrapped`**
резервирует под каждую такую ручку **полную ширину строки** и продолжает мерить
аддитивно. Итог — `max_rect` контента карточки раздувается примерно вдвое.
Снейл центрируется в доступной ширине (`available_size_before_wrap().x`) ⇒ центр
уезжает на край.

## Фикс
Убрать `horizontal_wrapped`; ручки — простой вертикальной стопкой (одна на строку),
их раскладывает обычный вертикальный layout карточки. Ширина считается от реального
контента, удвоения нет. См. `draw_mobile_snail_settings` в
[`resonator_panel.rs`](../../src/app/resonator_panel.rs) (и комментарий-предупреждение
прямо над функцией).

Заодно на мобиле убран внутренний подзаголовок спирали (`SpiralChart.subtitle = ""`),
т.к. он рисуется на одной строке с заголовком и на узком экране налезал.

## Правило на будущее (на что смотреть)
1. **Не вкладывай не-переносящий `ui.horizontal` внутрь `ui.horizontal_wrapped`.**
   Хочешь, чтобы блок переносился как единое целое — раскладывай блоки вертикально
   (стопкой) либо используй `egui::Grid`, а не «обёртку wrapped + вложенный horizontal».
2. **«Центр круга ровно на крае» = ровно ×2 по ширине.** Не лечи это центрированием/
   клампом высоты — ищи, кто раздул `max_rect`.
3. **Диагностируй послойно.** `available_size_before_wrap()`/`max_rect()` верны
   на верхних слоях и врут ниже ⇒ виновник — конкретный дочерний виджет между
   соседними замерами. Бинарный поиск по `alog` находит его за один билд.
4. **`set_max_width` на родителе — не фикс, если распирает дочерний виджет.**
   Родитель тут был корректен (376.7); клампить его бессмысленно. Чини источник.
5. Снейл рисуется в `painter_at(rect)` (клипается по `rect`), поэтому «вылезание»
   видно только если **сам `rect`** шире экрана — т.е. проблема всегда в аллокации,
   не в рисовании.
